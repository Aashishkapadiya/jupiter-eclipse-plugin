  • Release 2.1.913: September 13, 2004
    - New Features:
    • Supports the automatic reviewer and assigned to person filter.
      The "automatic" means that the filter in reviewer and assigned person is automatically changed by the reviewer ID selection in the review phase selection. The benefit of this is that the author can control the review issues shown in each phase by reviewer name and assigned to person. For example, given the automatic reviewer filter on in the individual phase, if the "foo" reviewer ID is selected in the phase selection, then review issues are filtered by the foo reviewer. if the "bar" reviewer ID is selected the same phase selection, then review issues are filtered by the bar reviewer.

  • Release 1.4.413: April 13, 2004
    - New Features:
    • Supports property page for each project. Now an author (team leader) of a review session is supposed to configure a review ID in the property page. You can see the property page by right clicking on a root project name in the "Package Explore" or "Navigator" view, selecting "Properties | Jupiter". To create new review ID, click "Add.." button to start "Review ID Creation" wizard. You can manage review IDs there such as modification and deletion too. Along with the movement of the configuration, the review phase is only three phases (i.e. Individual Review Phase, Team Review Phase, and Rework Phase). Don't forget to commit".jupiter" configuration file in the project root directory to the CVS. The former review "id" file was replaced by ".jupiter" file. Each review ID is to be stored in the ".jupiter" file. Note that if you are using Jupiter in a team, all team members are supposed to update this version. Otherwise, un-updated Jupiter can not recognize the previous review ID.
    - Bug Fixes:
    • Fixed the problem that the column width was not read so that the columns could not be loaded properly after review mode was changed.

    • Moved the configuration setting to the property page setting from the configuration phase menu in the main tool icon.
  • Release 1.4.330: Mar 30, 2004
    - New Features:
    • Supports the on-select project filter when "Where on-selected project is" is checked in the filter preference page. This facilitates you to switch the filter domains back and forth by the selected project root icon. To use this function, you can select the "Where on-selected project is" is checked in the filter preference page, then select a project root icon in the "Package Explore", for example.
    • Supports view preference page to specify a list of columns to be included or excluded, the order of the columns, and the column width. For example, you might want to change the order of column in the individual phase. You can go to the view preference page by selecting "Window | Preferences | Jupiter | View", then select "Individual phase" from pull down menu on the top, then select summary field and click the upper arrow icon to move up by one. The order will be changed from "Icon", "Type", "Summary" to "Icon", "Summary", "Type". Don't forget to click "OK" or "Apply" to apply this change.
    • Supports that the width of an column in the view is saved.
    • Supports that review session is specified by review ID. The review ID defines review session. It will also define reviewers and category items such as type, severity and so forth in the future. This main advantage is to be less likely to make users conflict in CVS.
      (1) Configuration- One author can define a review ID from Configuration Phase to specify Review ID and review ID file name. File name should be no space between characters .After the review ID defined the <review_ID>.id file will be created so that the file should be committed to the CVS.
      (2) Individual Review - Reviewers update CVS to get the id file, then select the Individual Review Phase to choose one of the review ID list. They can add issues as many as possible to be stored in the review ID related review file automatically and specify Type, Severity, Summary, and Description. After reviewing it, they should commit the ID related review file to the CVS.
      (3) Team Review - Moderator updates CVS to get all review files and review ID, then select the Team Review Phase to choose the review ID from the ID list. They can review as many as possible and set Resolution, Annotation, and AssignedTo field and update the modified issues. After the meeting done, one should commit all review files to CVS.
      (4) Rework - The author updates CVS to get al reviewed files, then select Rework Phase to choose the review ID from the ID list. One can fix the issues and change the Status to "resolved" and put comments or remainder to the Rework field. After the fixes, one should commit the modified review files to the CVS.
    - Bug Fixes:
    • Fixed the problem that the file path was not loaded to the Jupiter editor when no text was selected to add new Jupiter issue.

  • Release 1.4.212: Feb 12, 2004
    - New Features:
    • Supports Eclipse 3.0 M7.
    • Supports the sort by description, annotation, and revision.
    • Supports the review phase partially in the both main tool bar and Jupiter view tool bar: Config, Individual, Team, and Rework phase. You can see these phase selection in the tiny down triangle button just right to the purple jupiter icon. Each phase has own columns in Jupiter Issues view. This order and column width can be configurable in the config.xml file.
    • Supports XML configuration file (config.xml in the <install-eclipse-dir>/workspace/.metadata/.plugins/csdl.jupiter directory). This feature enables you to configure the order of each review phase and width of each column. Since <ColumnHeader> element will be read by the order of the description in the xml file, you can rearrange of the order. For example, if you want to assign the type followed by severity column, you can describe something like:
      <ColumnHeader name="Type" width="55" resizable="true"/>
<ColumnHeader name="Severity" width="50" resizable="true"/>
      Note that the name (as well as the name of the<Configuration>, and default of the <View>) MUST NOT be changed in this version. Otherwise it will break Jupiter. The "width" is the width of the column. The "resizable" is whether the column can be resizable or not". As you can see the first element of the <ColumnHeader> whose name is empty, and which is for the linked picture, its columns can not be resizable because of the "resizable=false". In the future, this configuration would be made through a Jupiter preference page.
    • Supports the automated incremental review on Team review phase when Enter button is pushed. This feature facilitate a moderator in a team to review each entry. For example, you can click the first issue in the top of the Jupiter review to start team review. After changing something such as resolution(e.g. from Unset to Valid-Needsfixing), you can just push "Enter" button to see the next listed issue in the Jupiter Issue view.
